# Liotan VPS firewall baseline

Цель: VPS не должен принимать прямые публичные подключения к Node.js API.
Node.js должен слушать только `127.0.0.1:3001`, а внешний доступ к API должен идти через Cloudflare Tunnel / nginx согласно production-схеме.

## Кодовая защита

В production сервер по умолчанию слушает:

```text
HOST=127.0.0.1
PORT=3001
```

Это задаётся в `server/config/env.js` и применяется в `server/server.js`.
Если `HOST` явно не указан, production не открывает Node.js на `0.0.0.0`.

## Что разрешать на firewall провайдера

Для текущей Tunnel-схемы самый безопасный базовый вариант:

```text
INBOUND:
- SSH 22: разрешить только с твоего доверенного IP, если он стабильный. Если IP динамический — не включать жёстко без запасного доступа.
- TCP 80/443: можно закрыть, если frontend/API реально идут только через Cloudflare Tunnel и прямой nginx-доступ не нужен.
- TCP 3001: закрыть для всех.
- Остальное: закрыть.

OUTBOUND:
- Разрешить, иначе cloudflared, MongoDB Atlas, R2, npm/apt и почта могут перестать работать.
```

## Что нельзя делать вслепую

Не закрывать SSH, пока нет подтверждённого доступа.
Не закрывать outbound, пока отдельно не расписаны адреса Cloudflare, MongoDB Atlas, R2, Resend и системных обновлений.
Не открывать публично `3001`: API должен быть доступен только через локальный origin/tunnel/proxy.

## Проверка на VPS

```bash
ss -lntp | grep -E ':3001|:8080|:80|:443|:22'
```

Ожидаемо для Node.js:

```text
127.0.0.1:3001
```

Не должно быть:

```text
0.0.0.0:3001
:::3001
```

Проверка health через production-домен:

```bash
curl -s https://api.liotan.com/health
```
